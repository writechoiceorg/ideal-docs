GET

This verb is used to read (or retrieve) a representation of a resource. In a successful request, GET returns a representation in JSON and an HTTP response code of 200 (OK). In the event of an error, it usually returns a 404 (NOT FOUND) or 400 (BAD REQUEST).

POST

The POST verb is most-often utilized to create new resources. In particular, it's used to create subordinate resources. When creating a new resource, the service associates the new resource with the parent, assigning an unique identifier. On successful creation, this verb returns HTTP status 201 (CREATED).

PUT

This verb is usually employed to update resources that have been previously created. The user sends a request body containing the newly-updated representation of the original resource. On a successful update, it returns a 200 status.

PATCH

PATCH is used for modifying capabilities. The request using this verb only needs to contain the changes to the resource, not the complete resource.

DELETE

DELETE is very straightforward. It is used to delete a resource identified by a unique identifier. On successful deletion, it returns a HTTP status 200 (OK) along with a response body.
